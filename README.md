## DayDot（个人离线事件记录）

DayDot 是一个**纯前端**、**零网络请求**的个人 Web App，用来记录不规律事件的日期，并快速查看“上次什么时候、已过去多少天”。数据默认只保存在浏览器本地（`localStorage`）。

---

## 本地运行（开发/自测）

在项目根目录启动静态服务即可：

```bash
python -m http.server 5173 --bind 0.0.0.0
```

打开：
- 电脑：`http://localhost:5173/`
- 手机（同一局域网）：`http://<你的电脑IP>:5173/`

说明：
- 开发态（localhost/局域网 IP）会**禁用 Service Worker**，避免缓存导致“改了看不到”。

---

## 数据存储与安全边界

- **存储位置**：浏览器 `localStorage`，key 为 `daydot`（见 `js/store.js`）。
- **数据范围**：仅文字（标签、日期、备注），无图片；不上传服务器。
- **备份文件**：导出 JSON 为明文，包含你的记录内容；请自行妥善保管（不要随意发群/网盘公开分享）。

---

## 导出 / 导入（备份）

入口：记录页底部的 **“数据备份”** 模块。

- **导出备份**：下载 `daydot-backup-*.json`
- **导入恢复**：
  - 导入前会**自动导出一份“导入前备份”**，防止误覆盖
  - 导入内容写入 `localStorage`

兼容性策略（当前取舍）：
- **只保证“records 不丢”**；未来版本即使标签结构调整，旧记录也会保留，最多表现为“未分类/标签为空”。

## 发版到 GitHub Pages（手机端如何更新）

项目包含 `sw.js`（离线缓存）。为了避免手机端长时间停留在旧版本，发版流程必须包含“更新 SW 版本号”这一步。

### 发版步骤（你每次更新都照做）

1) **发版前自动 bump `sw.js` 版本号**

```bash
node ./scripts/bump-sw-version.mjs
```

它会自动修改 `sw.js` 顶部的 `VERSION`，从而让手机端识别为“新版本”，刷新后拉取最新缓存。

2) 推送代码并发布到 GitHub Pages（按你的现有方式）

### 手机端更新方式（使用者视角）

- 打开 App 后若弹出“发现新版本/是否刷新”，点确认即可更新。
- 若没有弹窗但你确定已发版：**完全退出 PWA/浏览器标签页后重新打开**，通常会触发更新检测。
